以下是关于注释规范的详细解释和代码示例，帮助您理解每个要求：

---

### 1. **版权声明 (Copyright)**

在模块文件的开头使用 `#` 进行版权声明。

**示例**：
```python
# Copyright 2024, Company Name
# Licensed under the MIT License.
# Created by John Doe on 2024-11-30.
```

---

### 2. **声明注释 (Declaration)**

#### **2.1 模块头 (Module Header)**
使用文档字符串 (`docstring`) 描述模块的用途和注意事项。

**示例**：
```python
"""
This module provides utilities for data processing, including 
cleaning, transformation, and aggregation. 

注意：本模块仅支持 Python 3.7 及以上版本。
"""
```

#### **2.2 函数头 (Function Header)**

函数文档字符串需要详细说明功能、参数及返回值。

**示例**：
```python
def calculate_area(length, width):
    """
    计算矩形的面积。

    参数:
    length (float): 矩形的长度。
    width (float): 矩形的宽度。

    返回:
    float: 矩形的面积。
    """
    return length * width
```

#### **2.3 类头 (Class Header)**

类的文档字符串描述类的功能。

**示例**：
```python
class DataProcessor:
    """
    数据处理器类，用于清洗和转换数据。

    方法:
    - clean_data: 清洗数据。
    - transform_data: 转换数据格式。
    """
    pass
```

---

### 3. **语句注释 (Statement Comments)**

#### **3.1 if 语句的注释**  
在各分支处注释条件及功能（语句块少于 5 行可例外）。

**示例**：
```python
def categorize_score(score):
    """
    根据分数分类。

    参数:
    score (int): 分数。

    返回:
    str: 分类结果。
    """
    if score >= 90:  # 分数大于等于90为优秀
        return "优秀"
    elif score >= 60:  # 分数大于等于60为合格
        return "合格"
    else:  # 分数小于60为不合格
        return "不合格"
```

---

#### **3.2 for/while 循环的注释**

在循环头部说明循环条件和具体功能。

**示例**：
```python
# 遍历一个列表，计算其总和
total = 0
for num in [1, 2, 3, 4, 5]:
    total += num  # 累加当前数字
```

---

#### **3.3 行注释的位置**

行注释应放在代码行上方，而非行尾。

**示例**：
```python
# 计算矩形面积
area = calculate_area(5, 10)

# 错误写法（行尾注释不推荐）
area = calculate_area(5, 10)  # 计算矩形面积
```

---

#### **3.4 禁止屏蔽无用代码**

不允许用注释隐藏无用代码，应该直接删除。

**错误示例**：
```python
# def old_function():  # 无用代码，应该直接删除
#     pass
```

---

#### **3.5 优先使用中文注释**

除非团队明确要求使用英文，中文注释更直观易懂。

**示例**：
```python
# 计算圆的面积
def calculate_circle_area(radius):
    """
    根据半径计算圆的面积。
    """
    return 3.14159 * radius ** 2
```

---

### 总结规范

| **类型**         | **内容**                                                                                                                                 |
|------------------|-----------------------------------------------------------------------------------------------------------------------------------------|
| **版权声明**      | 使用 `#` 在模块头部进行版权及许可声明。                                                                                                    |
| **模块头**        | 使用 `"""docstring"""`，描述模块的用途及注意事项。                                                                                         |
| **函数头**        | 使用 `"""docstring"""`，描述函数功能、参数及返回值。                                                                                        |
| **类头**          | 使用 `"""docstring"""`，描述类的功能及方法。                                                                                               |
| **if 分支注释**    | 在条件判断的每个分支加注释，描述条件的意义和功能（少于 5 行可例外）。                                                                      |
| **循环注释**       | 在循环头部注释说明条件和目的。                                                                                                           |
| **行注释位置**     | 行注释放在代码上方，不放行尾。                                                                                                           |
| **无用代码处理**    | 禁止用注释屏蔽无用代码，应直接删除。                                                                                                     |
| **注释语言**       | 优先使用中文注释，除非团队明确要求使用英文。                                                                                              |

通过这些规范，注释将变得更加清晰、有条理，有助于团队合作和代码维护。
